15      /**
16       * From article: <blockquote>Notice that on line 171, the developer compares
17       * body against null. Unfortunately, the developer probably meant to make
18       * that comparison == instead of !=. In the case where the pointer is null,
19       * the code will skip over the assignment on line 172 and dereference the
20       * body variable on line 175. Oops. </blockquote>
21       * 
22       * Actually, that analysis is quite wrong. The problem wasn't that the
23       * developer meant to compare with ==. The problem is that the length check
24       * should have been inside the if (body != null) test.

28      public static String listing1(String body) {
29          if (body != null) {
30              body = body.trim();
31          }
32  
33          if (body.length() < 1) {
34              body = null;
35          }
36          return body;

60      /**
61       * From article: <blockquote> The struts code from the previous example has
62       * been slightly modified to introduce a data dependence between the value
63       * of body and the value of body_tracker. Notice that after the test of body
64       * against NULL, the value of body_tracker will be 5 if body is not NULL and
65       * 12 if body is NULL. As such, there's no longer a NULL dereference on line
66       * 177 because it's guarded by the check of body tracker. This example is
67       * simple enough, but may fool some simple analysis engines into reporting
68       * the defect where there really is no problem at all because there's no
69       * possible execution path that leads body to be dereferenced when NULL.
70       * </blockquote>
71       * 
72       */
73      public static String listing3(String body) {
74  
75          int body_tracker = 5;
76          if (body != null) {
77              body = body.trim();
78          } else {
79              body_tracker = 12;
80          }
81  
82          if (body_tracker != 12 && body.length() < 1) {
83              body = null;
84          }
85          return body;
86      }
